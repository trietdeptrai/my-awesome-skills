# Intake and approval guide

## Conversation contract

Ask only one question per turn. Give exactly three mutually exclusive options, with the recommended option first. A good question looks like:

1. **Apply the ideas in an original world — Recommended.** Best for active learning and a clear copyright boundary.
2. **Stay close to the book's setting.** Appropriate when the user owns or licensed the material.
3. **Create an abstract puzzle collection.** Useful when story would distract from the concepts.

Do not ask for information already present in the book, project files, or earlier answers. When a new answer opens another design branch, resolve that dependency before moving on.

## Decision order

Resolve decisions in this order because later choices depend on earlier ones:

1. Source availability and usable coverage.
2. Rights status: public domain, owned/licensed, or unconfirmed.
3. Fiction or nonfiction path.
4. Player age range and prior familiarity.
5. Primary outcome: learn, practice, reflect, or experience.
6. Desired duration; default to 30–60 minutes.
7. Tone and emotional intensity.
8. Existing project or new React/TypeScript project.
9. Constraints: deadline, devices, offline use, accessibility, or hosting.

If the user supplies several of these in one message, record them and ask only the next unresolved question.

## Gate 1: adaptation brief

Present:

- Source and coverage.
- Rights boundary.
- Target player.
- Player outcome.
- 30–60 minute scope unless changed.
- Originality boundary.
- Technical target.

Offer three ways to resolve any remaining high-impact decision. Ask for explicit approval before making the quest map.

## Gate 2: quest map

Present 5–7 rows by default with:

| Quest | Source anchor | Player verb | Mechanic | Consequence | Takeaway |
|---|---|---|---|---|---|

Check that the sequence builds understanding or dramatic pressure instead of merely following chapter order. Ask for explicit approval before implementing.

## Gate 3: playable direction

Build the smallest representative vertical slice: intro, one complete quest, feedback, takeaway, and saved progress. Show the slice and summarize:

- Interaction feel.
- Original visual direction.
- Accessibility behavior.
- Essential artwork proposed, if any.

Offer three concrete refinement directions with one recommendation. Complete the remaining quests only after approval.
